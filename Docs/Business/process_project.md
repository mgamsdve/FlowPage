# process_project.md
## Déroulement d’un projet Flowpage — de la prise de contact à la livraison

Ce document décrit le déroulement type d’un projet Flowpage.
Il sert de cadre interne pour garantir une expérience fluide, claire et professionnelle,
sans complexité inutile pour le client.

Flowpage privilégie un processus simple, rassurant et maîtrisé.

---

## 1. Prise de contact initiale

La prise de contact peut venir :
- d’un message de prospection envoyé par Flowpage
- d’un contact via le site

Dans tous les cas, le premier échange se fait par écrit.

Flowpage envoie :
- un message clair expliquant brièvement la démarche
- un questionnaire structuré (ex : Google Form)

Objectif :
- comprendre le contexte
- éviter un appel inutile
- laisser le prospect répondre à son rythme

---

## 2. Qualification du besoin

À ce stade, Flowpage cherche à comprendre :
- l’activité du client
- le rôle attendu du site
- les problèmes actuels (ou l’absence de site)
- le niveau de clarté déjà existant

La qualification reste volontairement **légère**.

Le but n’est pas encore de concevoir le site,
mais de vérifier que :
- le projet est pertinent
- le cadre est sain
- une collaboration est envisageable

---

## 3. Proposition de solution

Une fois le besoin global compris, Flowpage propose :
- une approche adaptée
- un type de site ou de pages
- un périmètre clair mais flexible

La proposition est envoyée sous forme :
- d’un message écrit structuré
- sans jargon
- sans document lourd

L’objectif est que le prospect se dise :
> “C’est clair, simple et rassurant.”

---

## 4. Accord et démarrage

Si le prospect est d’accord :

- un accord écrit simple (mail ou message) suffit
- aucun contrat lourd n’est imposé
- les conditions essentielles sont validées par écrit

Le projet démarre uniquement lorsque :
- l’accord est clair
- les attentes sont alignées
- le cadre est compris des deux côtés

---

## 5. Phase de conception et développement

Flowpage prend en charge la quasi-totalité du travail.

Principes :
- le client n’a pas à gérer la technique
- le processus doit rester simple pour lui
- Flowpage avance de manière autonome

Pendant le développement :
- une version de travail peut être montrée
- les échanges restent ponctuels
- le client n’est pas impliqué en continu

L’objectif est de respecter le temps et l’énergie du client.

---

## 6. Ajustements et retours

Les retours du client sont :
- encouragés
- pris en compte tant qu’ils restent cohérents

Flowpage reste volontairement souple :
- pas de nombre strict de rounds
- ajustements raisonnables inclus
- demandes excessives ou hors cadre discutées au cas par cas

Le rôle de Flowpage est aussi de guider et de cadrer.

---

## 7. Livraison du site

La livraison comprend :
- le site mis en ligne
- une explication claire par écrit
- la validation finale du rendu

Par défaut :
- le client n’a pas accès au code
- la gestion technique reste chez Flowpage

Un accès ou un CMS peut être envisagé uniquement si :
- le besoin est réel
- cela ne nuit pas à la clarté ou à la qualité

---

## 8. Après la livraison

Après la livraison, Flowpage propose :
- un service de maintenance
- hébergement
- corrections techniques
- ajustements mineurs

La maintenance est proposée sous forme :
- d’un abonnement mensuel
- sans obligation
- avec un cadre clair

Le client reste libre de son choix.

---

## 9. Arrêt ou refus d’un projet

Flowpage se réserve le droit de :
- refuser un projet
- stopper une collaboration

Notamment si :
- le projet stagne excessivement
- le client ne communique plus
- le cadre n’est pas respecté
- le sérieux ou le paiement pose problème

La qualité de la relation prime sur la finalisation à tout prix.

---

## 10. Principe non négociable

Un projet Flowpage se passe bien si :
- le site avance
- le client communique honnêtement
- le cadre est respecté

Un projet est stoppé si :
- il n’avance plus
- le client n’est pas sérieux
- les limites ne sont pas respectées

---

## 11. Rôle de ce document

Ce document sert à :
- structurer le travail
- rassurer les prospects
- éviter les improvisations
- maintenir une posture professionnelle
- garantir une expérience simple et saine
